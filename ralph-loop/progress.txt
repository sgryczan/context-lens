# Progress Log — AWS Bedrock Support for Context Lens
Started: 2026-03-02

## Architecture Decision
Bedrock uses SigV4 request signing tied to the destination host, so reverse proxy (ANTHROPIC_BASE_URL override) is NOT viable — the proxy would receive a request signed for localhost, which Bedrock rejects. Use MITM interception instead (same pattern as Codex, Cline, OpenCode). Bedrock's invoke API uses the same request/response body format as Anthropic Messages API — reuse 'anthropic-messages' apiFormat, do NOT create a new one.

## Bedrock API Reference
- Invoke: POST /model/{modelId}/invoke (same body as Anthropic Messages API)
- Converse: POST /model/{modelId}/converse
- Stream: POST /model/{modelId}/invoke-with-response-stream (returns application/vnd.amazon.eventstream)
- Auth: AWS SigV4 — authorization header starts with 'AWS4-HMAC-SHA256'
- Regional endpoints: bedrock-runtime.{region}.amazonaws.com
- Model IDs: anthropic.claude-3-5-sonnet-20241022-v2:0, us.anthropic.claude-sonnet-4-20250514-v1:0
- Normalization: strip region prefix (us.), vendor prefix (anthropic.), version suffix (-v1:0)
- EventStream binary format: [4B total_len][4B headers_len][4B prelude_crc][headers][payload][4B message_crc]
- Env vars that signal Bedrock usage: CLAUDE_CODE_USE_BEDROCK=1, ANTHROPIC_BEDROCK_BASE_URL
